# Тестовый отдел продаж

Интерактивная песочница ИИ-автоматизации продаж: посетитель говорит/пишет ИИ-менеджеру и
в реальном времени видит, как разговор заносится в CRM, формируется саммари и считается
экономика. Работает как лидген на сайте и как демо-инструмент на встречах.

Архитектура — в [ARCHITECTURE.md](./ARCHITECTURE.md).

## Запуск (разработка)

```bash
npm install
npm run dev:backend     # http://localhost:8080
npm run dev:frontend    # http://localhost:5173
```

Без ключей Dasha фронт стартует в режиме **simulator** — проигрывает скриптованный
диалог и показывает весь демо-цикл локально.

## Конфигурация

Скопируй `backend/.env.example` → `backend/.env` и заполни (см. комментарии в файле).
Без `DASHA_API_KEY` бэкенд отдаёт сессии в режиме `simulator`.

## Структура

```
backend/    Fastify: минт токенов Dasha, лимиты, лиды → Supabase, ROI-конфиг
frontend/   React SPA: CRM-макет, виджеты, ROI-панель, Dasha ws-клиент + симулятор
shared/     общий конфиг ниш и типы
```

## Деплой

pm2 + Caddy на основном сервере, поддомен `demo.flowsmart.ru`. См. ARCHITECTURE.md.
